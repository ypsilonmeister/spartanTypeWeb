# **プロジェクト要件定義書：SpartanType Web**

作成日: 2026年6月16日

## **1\. プロジェクト概要**

本プロジェクトは、長年染み付いた標準的な行配置（Row Staggered）キーボードの「独自のタイピングの癖」を完全に解体し、標準的なキーボード（フルサイズ、TKLなど）および36キーなどの極小配列かつ列配置（Column Staggered）のキーボード環境に最適化された「正しい運指」を身体に再学習させるための、ブラウザ完結型・インストール不要のスパルタ式運指矯正Webアプリケーション「SpartanType Web」の要件を定義するものである。  
従来のタイピングソフトのように単に「正しいキーを押したか」を判定するのではなく、「正しい指で押したか」をカメラによる画像認識を用いて厳密に評価する。さらに、入力テキストに「植物分類学」の階層構造（ローマ字入力）を取り入れることで、退屈な反復練習を排し、知的好奇心とパターンの発見を刺激する先進的な学習体験を提供する。

## **2\. ユーザー体験（UX）のコアバリュー**

### **2.1 「速度」より「正確な型」の絶対視**

どれほど高速に入力できたとしても、選択したキーボードレイアウト（行配置/列配置）ごとに指定された指以外の不適切な指（例：中指で押すべきキーを人差し指で手を伸ばして押すなど）で打鍵された場合は、入力の価値をゼロ（またはエラー）として扱う。自己流の妥協を一切許さない厳格なトレーニング環境を提供する。

### **2.2 事後解析による「気づき」の提供（ゼロ・レイテンシの追求）**

タイピング中リアルタイムに重い画像認識処理を挟むと、キーボード本来の軽快な打鍵感（スタッカート）やタイピングのリズムが阻害される。そのため、入力中は映像の録画と打鍵ログの記録（タイムスタンプ）に徹し、セッション終了後に非同期で徹底的な「ビデオ判定・フォーム解析」を行う。これにより、入力中のストレスフリーな打鍵感と、終了後のディープなフィードバックを両立させる。

### **2.3 知的好奇心と視覚的報酬の連動**

無味乾燥な文字列のタイピングではなく、意味のある系統的な階層構造を入力させる。正しい運指で入力が成功するごとに、画面上の論理的なツリーや美しいグラフィックスが展開していき、構造をマッピングしていくカタルシスをユーザーに与える。

## **3\. キーフィーチャー（主要機能要件）**

### **3.1 ズルを許さない「スパルタ運指判定」と減点システム**

カメラを通じてユーザーの手および指先の骨格を監視し、選択したキーボードレイアウト（標準QWERTYなどの行配置、または36キーなどの列配置）ごとに定義された「担当指」のマッピングと、実際に打鍵した「動作指」を厳密に照合する。本来の担当外の指が動いてキーが押されたことを検知した場合、容赦なくエラーとして判定し、スコアを減点する。画面上に監視ステータスを明示することで、無意識に古い癖に逃げることを心理的に抑制する。

### **3.2 植物分類ツリーのドリルダウン入力（ローマ字メイン）**

練習用の辞書（テキストデータ）には、植物分類学の階層構造（科・属・種）を採用し、まずは日本語のローマ字入力をメインとする。マクロな分類からミクロな個別種へとドリルダウンしていく形式で入力させる。

| 分類階層 | 表示名 | 入力テキスト（ローマ字）   |
| :---- | :---- | :---- |
| 1\. 科 (Family) | キク科 | KIKUKA |
| 2\. 属 (Genus) | タンポポ属 | TANPOPOZOKU |
| 3\. 種 (Species) | セイヨウタンポポ | SEIYOUTANPOPO |

正しい運指で入力が完了するごとに、背景に植物の分類ツリー（樹形図）が美しく枝分かれしながら描画され、自身のタイピングの進捗が論理的な構造の可視化と直結する仕様とする。

### **3.3 振り返り（事後解析）ダッシュボード**

セッション終了後、記録された動画と打鍵タイムスタンプを突き合わせ、詳細なレポート画面を生成する。

* **エラーの可視化:** 「どのキーを押した時に、どの指が、どの方向へ本来の軌道からズレて（越境して）しまったか」をヒートマップやタイムラインで再現する。  
* **癖のラベリング（分類学アプローチ）:** 「人差し指・過干渉型」「小指・サボり型」など、ユーザー固有の誤打鍵パターンの傾向を、まるで植物の亜種を分類するように客観的に特定・ラベリングして蓄積する。

### **3.4 マルチアングル監視（拡張要件）**

1台のメインカメラだけでは、指同士の重なり（オクルージョン）によって誤判定が起きる可能性があるため、スマートフォン等の別デバイスをサブカメラとしてシームレスに連携させる機能を備える。例えば、メインカメラを真上（俯瞰）に配置し、スマートフォンを横（サイド）に配置することで、指の3次元的な動きやホームポジションからの微妙な浮き沈みを完全に捕捉する環境をユーザー自身に構築させる。

### **3.5 キーボードレイアウトの動的マッピングと4点キャリブレーション**

カメラ映像とキーボード上のキー座標を紐づけるため、射影変換（ホモグラフィ）を用いた柔軟なキャリブレーションシステムを採用する。

* **レイアウト定義の読み込み:**
  * **カスタムキーボード:** 自作キーボード等のVial（`https://vial.rocks/`）やKeyboard Layout Editor（KLE）から書き出されたレイアウトJSON（キーの位置・サイズ情報を含む）を読み込む。
  * **標準キーボード:** 一般的なJIS配列やUS配列のキーボードについては、あらかじめ標準的なプリセットを用意する。
* **初期キャリブレーション（4点）:**
  * アプリの指示に従い、キーボードの「四隅の特定のキー」（例：左上「Esc/1」, 右上「BackSpace/¥」, 左下「Ctrl/Shift」, 右下「Ctrl/矢印」）を一本の指で順番に打鍵する。
  * 打鍵された瞬間の打鍵イベントのタイムスタンプと、その瞬間にカメラから検出した指先（MediaPipe Hands）のピクセル座標をペアとして記録する。
  * 記録した4点の座標ペアから射影変換行列（ホモグラフィ行列）を算出し、レイアウト定義上の任意のキー座標`(x,y)`をカメラ画像上のピクセル座標`(px,py)`に変換・マッピングする。これにより、カメラの傾きによるパースの歪みやキーボードの傾きを数学的に補正する。
* **漸進的・オンデマンドな精度向上（フィードバック型補正）:**
  * **初期ハードルの引き下げ:** 初回から完璧な4点キャリブレーションを強制しなくても、おおまかなプリセット位置で即座に練習を開始できるようにする。
  * **誤判定キーのピンポイント補正:** 事後解析の結果、「判定の信頼度が低いキー」や「誤判定の疑いがあるキー」がある場合、フィードバック画面で「このキーをもう一度指一本で押してキャリブレーションしてください」とユーザーに促す。
  * **最小二乗ホモグラフィによるフィッティング:** キャリブレーションデータが5点、6点と増えるにつれ、最小二乗法（または RANSAC 等のロバスト推定アルゴリズム）を用いて射影変換行列を再計算・最適化する。これにより、使用を重ねるほどそのユーザーの環境（手の置き方、キーボード位置）へのマッピング精度が段階的に極限まで高まっていく。

## **4\. 非機能要件（セキュリティとプライバシー）**

本システムは、家庭内やプライベートな空間、また子供の学習環境でも安心して利用できるよう、カメラ映像や入力ログのサーバーへのアップロードを一切行わない完全な「クライアントサイド（ローカルブラウザ）完結型」の設計とする。外部へのデータ流出リスクを根本から排除し、完全にスタンドアロンの解析ツールとして動作させる。
